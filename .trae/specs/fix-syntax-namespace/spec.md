# 修复与规范化代码文件 Spec

## Why
代码库在演进或历史修改中，`src` 目录下的代码文件可能存在了语法错误、拼写错误、错别字、乱码、格式混乱或内容缺失。同时，因为老版本的架构 (`old_core` 目录) 未使用命名空间 (namespace)，现有的 `src` 目录下的代码在迁移时可能遗漏或错误声明了命名空间。
我们需要严格依据老版本的逻辑人工逐一比对并修复这些文件，添加并规范化命名空间，以防止历史代码损坏再次发生，确保所有代码都可以正确执行且无语义冲突。

## What Changes
- 逐文件（`src` 目录下所有文件）比对老版本（`old_core` 目录），并人工修复所有语法错误、拼写错误、注释乱码及缺失内容。
- 逐文件检查 PHP 类是否正确定义并使用了命名空间。若缺失，则依据当前的目录结构与老版本逻辑人工补充恰当的命名空间（如 `namespace Anon\Modules\Http;`）。
- 每完成一个文件的修复，在本地环境立即运行语法编译/检查（如 `php -l` 等），确保该文件没有任何报错后再继续下一个文件。
- **BREAKING**: 全程严禁使用终端批量命令、格式化工具或自动化替换脚本，必须确保是人工逐个文件逐字符进行确认和修改。

## Impact
- Affected specs: 系统的命名空间规范和代码格式正确性。
- Affected code: `d:\Codes\Anon\src\` 目录及其子目录下的所有文件。

## ADDED Requirements
### Requirement: 强制的命名空间规范
系统中的所有 PHP 类必须严格包含与其路径对应的 namespace 声明，确保旧版全局类的逻辑在有命名空间的结构下依旧有效且不发生命名冲突。

#### Scenario: 成功声明并修复
- **WHEN** 人工打开并修复 `src/Modules/Http/Router.php`
- **THEN** 发现存在拼写错误并修正，并且在文件顶部加上/确认存在了正确的 `namespace Anon\Modules\Http;` 声明，且保存后通过 `php -l` 测试。

## MODIFIED Requirements
### Requirement: 历史代码修复还原
所有的文件注释、代码逻辑在迁移过程中发生的乱码或内容丢失，必须以 `old_core` 中的对应文件为基准，进行人工恢复，确保业务逻辑不变。

## REMOVED Requirements
### Requirement: 自动化格式或替换
**Reason**: 自动化替换可能会掩盖或引发未知的破坏性变更。
**Migration**: 彻底废弃所有批量脚本操作，全部转为人工逐一文件比对处理。
